## 📌 Executive Summary
A highly automated Dev pipeline engine designed to ingest raw container and filesystem vulnerability scans, programmatically map complex JSON schemas, and aggressively filter out theoretical noise. 

By cross-referencing scanner exhaust with live federal threat intelligence (CISA KEV) and predictive mathematical models (FIRST EPSS), this engine transforms massive vulnerability data into a prioritized, actionable patching dashboard. 

### **📊 Quantifiable Impact**
* **Alert Fatigue Reduction:** Demonstrated an **88.8% reduction** in vulnerability noise during honeypot testing (reducing 9 critical/high CVEs to 1 genuinely exploitable threat).
* **Mean Time to Remediate (MTTR) Optimization:** Eliminates manual triage by instantly isolating vulnerabilities that are actively being weaponized by threat actors in the wild.
* **Data-Driven Prioritization:** Replaces static CVSS scores with dynamic, real-time exploit probability percentages (EPSS), ensuring engineering teams patch the most critical risks first.

---

## ⚙️ System Architecture & Data Flow

1. **Enterprise Ingestion:** Maps and drills through heavily nested JSON arrays generated by industry-standard cloud security scanners (e.g., Aqua Trivy).
2. **Threat Intelligence Correlation:** Pings the US Cybersecurity and Infrastructure Security Agency (CISA) Known Exploited Vulnerabilities catalog to perform binary filtering (Exploited in the wild vs. Theoretical).
3. **Predictive Analytics:** Queries the Forum of Incident Response and Security Teams (FIRST) API to assign a 30-day exploit probability score.
4. **Actionable Output:** Renders a mathematically sorted CLI dashboard for immediate patching triage.

---

## 💻 Proof of Concept: Modern Honeypot Evaluation
To validate the engine, a localized Python environment was populated with historically vulnerable dependencies (`Pillow==9.0.0`, `urllib3==1.26.4`). 

While the baseline scanner detected multiple high-severity flaws, the RBVM engine successfully filtered the noise to highlight the single most pressing threat: **CVE-2023-4863** (WebP Heap Buffer Overflow), calculating a **93.3% probability** of exploitation.
